The Interaction Pattern — Outcomes, Not Steps

DO
  • Describe outcomes

    "Create a README with installation instructions, usage examples, and API docs"

  • Give context

    "This is a TypeScript library for parsing CSV files with streaming support"

  • Trust iteration

    "Fix the failing tests" — let it diagnose and solve

DON'T
  • Micromanage steps

    "First read file X, then find line 42, then change the word 'foo' to 'bar'"

  • Assume limitations

    "You probably can't do this, but..." — it often can!

  • Interrupt mid-task

    Stopping before it finishes iterating on errors

Think collaborator, not command-line tool. Claude Code works best when you describe what you want to achieve, not how to achieve it.
