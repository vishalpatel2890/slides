Trust & Control — When to Guide vs Let It Run

High Trust

Let it run with minimal oversight

  • Refactoring existing code patterns
  • Writing documentation and comments
  • Generating unit tests for existing code
  • Exploring and explaining codebases
  • Creating boilerplate and scaffolding
Review Needed

Check before approving

  • Deleting files or directories
  • Calling external APIs or services
  • Modifying deployment configurations
  • Installing or removing dependencies
  • Git operations (push, merge, rebase)
Intervention Patterns — How to Course-Correct
Ctrl+C to cancel operations mid-stream
"Stop and try a different approach" — redirect mid-task
"Undo that last change" — revert specific actions
