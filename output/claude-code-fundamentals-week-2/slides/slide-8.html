Anthropic's Guide

Prompt Best Practices

First-principles techniques that dramatically improve output quality and reduce iterations.

Based on Anthropic Docs
Claude Code
01

Be Specific

Vague prompts get vague results. Specify exactly what you want, including format, length, and style.

"Add a login button" → "Add a login button to the navbar that redirects to /auth/login"
02

Provide Context

Include background, constraints, and goals. Claude can't read your mind — tell it what matters.

Use @file to inject relevant code, explain the architecture, mention constraints
03

Break Down Complex Tasks

Large tasks benefit from step-by-step decomposition. Smaller steps = clearer outputs.

"Build auth" → "First, create the User model. Then, add the login endpoint..."
04

Use Examples

Show, don't just tell. Examples clarify expectations better than descriptions.

"Format the output like this: { name: string, age: number }"
05

Iterate Deliberately

Refine based on outputs. Each iteration should add specific feedback, not just "try again."

"Good start, but make the error messages more user-friendly and add validation"
06

State Constraints

Tell Claude what NOT to do. Constraints prevent unwanted approaches.

"Don't use any external libraries" or "Keep the function under 50 lines"
